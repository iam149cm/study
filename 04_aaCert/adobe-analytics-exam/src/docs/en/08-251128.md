#### ‚úåüèª Segmentation  
- **Definition**: A method to filter visitors, visits, or hits that meet specific conditions.  
- **Segment Containers**  
  - Hit: filters individual page views or events  
  - Visit: includes all hits within a single session  
  - Visitor: applies to the user‚Äôs entire history  
- Nested segments can be combined for complex filtering logic.  
- ref link : https://experienceleague.adobe.com/en/docs/analytics/components/segmentation/segmentation-workflow/seg-workflow  

#### üëâüèª Calculated Metrics  
- **Definition**: Metrics created by combining existing metrics or formulas to form new KPIs.  
- Example: Conversion Rate = (Orders / Visits) √ó 100  
- Supports arithmetic operations, segmentation, and visualization-level calculations.  
- ref link : https://experienceleague.adobe.com/en/docs/analytics/components/calculated-metrics/cm-overview  

#### üß≠ Attribution Models  
- **Definition**: Methods for assigning credit for a conversion across multiple touchpoints.  
- Major models:  
  - **Last Touch** ‚Äì assigns 100% credit to the last interaction  
  - **First Touch** ‚Äì assigns 100% credit to the first interaction  
  - **Linear** ‚Äì distributes equal credit to all touchpoints  
  - **Time Decay** ‚Äì gives more weight to touchpoints closer in time to conversion  
- ref link : https://experienceleague.adobe.com/en/docs/analytics/analyze/analysis-workspace/attribution/models

### üìù Practical Points
- **Breakdown Visualization**  
  - Breaks down one dimension by another (e.g., Device Type ‚Üí Product Category).  
  - Useful with Segments for analyzing user behavior paths.  

- **Flow Visualization**  
  - Visually displays user navigation paths (Entry ‚Üí Interaction ‚Üí Exit).  
  - Forward Flow shows next steps; Backward Flow reveals entry sources.  
  - Real-world use: Funnel analysis (Signup ‚Üí Cart ‚Üí Purchase).  

- **Workspace Tips**
  - Use Custom Panels to organize dashboards per project.  
  - Combine Calculated Metrics and Segments for dynamic KPI comparison.  
  - Choose appropriate visualizations: Freeform Table, Flow, Fallout, Summary Number, etc.

### üéØ Practice Questions

**Q1.** What is the main advantage of using Analysis Workspace compared to traditional reports?  

    A. It allows drag-and-drop visualization and custom analysis  
    B. It only supports fixed dashboards  
    C. It cannot use calculated metrics  
    D. It replaces Adobe Launch for implementation  

**Answer: A**  
**Explanation:** Workspace enables flexible drag-and-drop analysis and visual customization beyond static reports.  

---

**Q2.** In which container should a segment be created to filter users who made a purchase across multiple visits?  

    A. Hit  
    B. Visit  
    C. Visitor  
    D. Session  

**Answer: C**  
**Explanation:** The Visitor container applies across multiple sessions and identifies users based on their complete history.  

---

**Q3.** Which Attribution Model gives equal credit to all touchpoints in the conversion path?  

    A. First Touch  
    B. Linear  
    C. Time Decay  
    D. Last Touch  

**Answer: B**  
**Explanation:** The Linear model distributes equal credit to every touchpoint in the user‚Äôs journey.  

---

**Q4.** In Analysis Workspace, what does a Flow visualization show?  

    A. Comparison between KPIs over time  
    B. Pathing of user navigation between pages or actions  
    C. Real-time data feed  
    D. Attribution weight per channel  

**Answer: B**  
**Explanation:** A Flow visualization displays how users move forward and backward through pages or actions.
