### ‚úÖ Mini Concept Study: Data Layer & Launch Rule Debugging

#### ‚òùüèª What is a Data Layer?
- **Definition**: A JavaScript object used to structurally manage the data to be collected on a website
- It is the core structure of any Tag Management System (TMS) such as Adobe Launch and Google Tag Manager
- Usually declared as a global object like `digitalData` or `dataLayer` to store user behavior, page info, and event info
- ref link : https://experienceleague.adobe.com/en/docs/analytics/implementation/prepare/data-layer

#### ‚úåüèª digitalData / dataLayer structure
- digitalData (based on W3C standard)
  ```js
  digitalData = {
    page: { pageName: "home" },
    user: { loginStatus: "guest" },
    product: [{ id: "1234", name: "Shoe" }]
  };
  ```
- dataLayer (Google GTM style)
  ```js
  window.dataLayer.push({
    event: "purchase",
    ecommerce: { value: 100000, currency: "KRW" }
  });
  ```
- In Launch, you extract only the required values with Data Elements and send them as eVars/props/events
- ref link : https://experienceleague.adobe.com/en/docs/analytics/implementation/launch/layer-to-elements

### üìù Practical Points
#### 1Ô∏è‚É£ Launch Rule firing conditions
- `Event Type` (trigger): Window Loaded, DOM Ready, Direct Call, Data Element Changed, etc.
- `Condition`: Control whether a Rule runs based on specific pages/data values
- `Action`: Execute Analytics calls, set cookies, run JS, etc.

#### 2Ô∏è‚É£ Launch Rule execution order (Sequence)
- Page Load phase: `Library Loaded (Page Top)` ‚Üí `DOM Ready` ‚Üí `Window Loaded`
- User interaction phase: Clicks, scrolls, and custom events trigger Direct Call events
- Asynchronous data updates: when `dataLayer.push()` occurs, you can re-run Rules via the `Data Element Changed` trigger
- ref link : https://experienceleague.adobe.com/en/docs/analytics-learn/tutorials/implementation/experience-platform-tags/using-a-data-layer-to-set-page-name-and-other-variables-via-tags

### üéØ Practice Questions

**Q1.** What is the main purpose of a Data Layer in Adobe Launch?

    A. To directly send analytics hits  
    B. To store structured information used by tags and rules  
    C. To generate HTML elements dynamically  
    D. To control ECID expiration  

**Answer: B**  
**Explanation:** The Data Layer stores data in a structured way for tags and rules to reference.

---

**Q2.** Which event type in Adobe Launch triggers when a variable in the Data Layer changes?

    A. Window Loaded  
    B. Direct Call  
    C. Data Element Changed  
    D. DOM Ready  


**Answer: C**  
**Explanation:** The Data Element Changed event re-runs a Rule when the dataLayer value changes.

---

**Q3.** Which tool is best for checking the parameters sent in an Adobe Analytics tracking call?

    A. Omnibug  
    B. Network ‚Üí Console Log  
    C. Privacy Service  
    D. Data Element Editor  

**Answer: A**  
**Explanation:** Omnibug visually displays the parameters of Adobe Analytics requests.
